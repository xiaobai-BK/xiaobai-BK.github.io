读取，修改和写入元数据

从数据库读取文档元数据和向数据库读取文档元数据与读取和写入文档内容非常相似。每个方法都需要在上调用方法com.marklogic.client.document.DocumentManager。元数据的Handles可以是DocumentMetadataHandle用于修改POJO中的元数据的Handles，也可以是原始XML或JSON。

文件中资料

枚举DocumentManager.Metadata枚举元数据类别（包括ALL）。以下是此枚举涵盖的元数据类型：

  1. COLLECTIONS：文档集合，这是在数据库中组织文档的一种非分层方式。
  2. METADATAVALUES：键值元数据，有时也称为元数据字段。
  3. PERMISSIONS：文档权限。
  4. PROPERTIES：文档属性。与文档关联的属性值对。
  5. QUALITY：文档搜索质量。帮助确定最优质的文档。

修补文档和元数据的基本步骤

对于JSON和XML文档，您可以使用com.marklogic.client.document.DocumentPatchBuilder来修补仅内容，内容加元数据或仅元数据。对于所有文档类型，您只能使用com.marklogic.client.document.DocumentMetadataPatchBuilder来修补元数据。一个DocumentPatchBuilder也是DocumentMetadataPatchBuilder。使用DocumentManager诸如JSONDocumentManager或的子类GenericDocumentManager来创建补丁构建器。

当您在同一补丁中组合内容和元数据更新时，补丁格式（XML或JSON）必须与补丁文档的内容类型匹配。
